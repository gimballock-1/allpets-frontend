# Security Policy

## Reporting a Vulnerability

Please **do not** open a public issue, pull request, or discussion for security
vulnerabilities — that would disclose the issue before a fix is available.

Instead, report privately through GitHub's **Private Vulnerability Reporting**:

1. Go to the [**Security** tab](https://github.com/gimballock-1/allpets-frontend/security)
   of this repository.
2. Click **"Report a vulnerability"**
   ([direct link](https://github.com/gimballock-1/allpets-frontend/security/advisories/new)).

This routes your report privately to the maintainers. If you cannot use that
channel, you may instead request a secure contact by opening a minimal issue that
contains **no vulnerability details**.

## What to include

- A description of the issue and its impact.
- Steps to reproduce (proof-of-concept, affected route/component, version/commit).
- Any suggested remediation, if known.

## Our commitment

- We aim to acknowledge your report within **5 business days**.
- We will keep you informed as we investigate and remediate.
- We will credit reporters in the advisory unless you prefer to remain anonymous.

## Supported versions

This project is under active development. Only the latest `main` branch is
supported; please report issues against the current `main`.
